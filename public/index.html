      Specification

      Create a comic book tag browser using the provided data.

      The tag list
      1. Render a list of all the tags from all the comic books into the '.tag-list' ul.
      2. There should not be duplicate tags
      3. The list should be alphabetized
      The Book List
      1. When a tag is selected (clicked), render the list of books with the matching tag into the '.matching-items-list' ul
      The Selected Book Data
      1. When a book is selected (clicked), render the data for that book into the '.selected-item' div
      2. The authors list is represented as an array, please display the authors as a comma-separated list
      The Clear Button
      1. When the clear button is clicked, clear the selected tag, clear the book list, and clear all book data. Essentially, reset the widget.

      Dependencies

      1. Bulma is included on the page and provides basic styling. You may add any new styles you need or remove this dependency if desired.
      2. We typically use Vue.js or React on most projects, however it is important to be familiar with plain vanilla javascript.
        Do not add additional libraries to the page (jQuery, React, Angular, Vue, etc).

      What are we looking for?

      1. How do you work with a dataset?
      2. How do you handle templating/dom manipulation without a framework?
      3. What strategies do you employ for event binding?
      4. How do you organize a class containing the different methods for performing these tasks?

      Some Tips

      1. Scroll down and take a look at the sample video.
      2. Check out the README in this repo.
      3. Feel free to add, remove, or modify new markup and styles as you see fit, in addition to the javascript in app.js.
      4. The <script> tag that instantiates the app is at the bottom of index.html
      5. There are two books with the same name, one having an alternate cover: Preacher #1. You will want to base your event handling on the UUIDs to differentiate between these. (perhaps using something like a data- attribute)
      6. It will probably create more complexity to try to break this into sub-components. I wouldn't recommend it.
      7. Do not worry about browser support outside of the latest version of chrome.

      Bonus

      1. It doesn't have to be pretty, but bonus points for doing some interesting styling or css animations
      2. Bonus for rendering semantically correct markup (anchors, etc)
      3. Bonus: Render "No Tag Selected" and "No Book Selected" text when there is not a selected tag/book.
      4. Bonus: Disable the "clear" button when there is not a selected tag/book
